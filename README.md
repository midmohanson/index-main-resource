# index-main-resource

这个仓库用于归档和发布多个独立 HTML 页面。页面内容涵盖不同主题和用途，统一存放在此，便于访问与维护。

## 项目简介

本仓库的主要目的是提供一个集中存放 HTML 页面的位置。每个页面都是独立的，不依赖特定的服务器环境，可以直接通过浏览器打开或部署到静态托管服务上使用。

## 目录说明

```
/
├── index.html          # 仓库入口页面，通常包含导航或说明
├── pages/              # 存放各独立 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源（CSS、JavaScript、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：仓库的主页，可以作为访问其他页面的入口。
- `pages/`：所有独立 HTML 页面放在此目录下，每个文件对应一个页面。
- `assets/`：存放页面共用的样式、脚本或图片资源。

## 页面归档说明

- 每个页面均为独立的 HTML 文件，可单独使用。
- 页面之间无强依赖关系，但可能共享 `assets/` 下的资源。
- 文件名应体现页面内容或用途，避免使用无意义命名。
- 建议在页面内部添加必要的说明或注释，方便后续维护。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的链接（如有需要）。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源路径正确。
- 删除页面：移除 `pages/` 下的文件，并清理 `index.html` 中的相关链接。
- 资源管理：公共资源应放在 `assets/` 目录下，避免在每个页面中重复引入。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 也可将整个仓库部署到任意静态托管服务，获得公开访问链接。
